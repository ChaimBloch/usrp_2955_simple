NI-USRP Simple Streaming with USRP-2945/2955

The NI-USRP Simple Streaming sample project demonstrates the following tasks with the USRP-2945/2955:

  • Streaming data from the input port of the USRP-2945/2955 device to the host VI

USRP-2945/2955 Connectors

Throughout this document, the term input port refers to the RF0 RX1, RF0 RX2, RF1 RX1, and RF1 RX2 SMA connectors on the device front panel.

System Requirements

  • LabVIEW 2016 Full or Professional Development System.
  • NI USRP RIO Instrument Design Libraries.
  • (Optional) LabVIEW FPGA Module 2016 to view, open, and edit the included FPGA VI. If you do not have the LabVIEW FPGA Module installed, you can still run this sample project. However, the Project Explorer window shows only the host VIs.

Features

  • Configures an input data stream to the host, other FPGAs, or inside the FPGA itself.
  • Applies frequency shifting on the FPGA in order to fully utilize the superheterodyne architecture of the USRP-2945/2955.
  • Uses decimation on the FPGA to reduce data sent to the host, which can prevent buffer overflow.
  • Supports both continuous and finite acquisitions.

Overview

This sample project contains two host VIs, which run on the host computer, and one FPGA VI, which runs on the FPGA. Each host VI configures the device input and output ports and configures a data stream between the host VI and the USRP-2945/2955.

The sample project performs the following tasks:

  • Streaming data from the device to the host
    1. The FPGA VI acquires data from the input port, processes the data, and writes it to the target-to-host DMA FIFO. You can see this FIFO in the Project Explorer window at My Computer»<Device Name>»Resources»FIFOs»rx 0.host fifo.
    2. The host VI reads from the target-to-host DMA FIFO and displays the data on the host VI front panel.

The input stream uses a Start Trigger to begin streaming data acquired from the input port. This stream can operate in the following modes:

  • Continuous—The FPGA streams data to the host until you stop the application. If the host cannot accept data quickly enough, the DMA FIFO fills too quickly, and the FPGA reports a buffer overflow.
  • Finite—The FPGA periodically sends a block of samples to the host. Specify the size of each block in the Number of Samples host VI front panel control. A block of data is sent when the Start Trigger is asserted.

    Set the mode using the Acquisition Mode host VI front panel control.

Host VIs

You can view the host VIs in the Project Explorer window by expanding My Computer. The following host VIs are included:

  • Rx Streaming (Host).vi—Streams data from the input ports of a single device to the host VI.
  • Multi-Device Rx Streaming Time (Four Channels) (Host).vi—Streams data from the input ports of two devices to the host VI.

For more information about using and connecting multiple USRP-2945/2955 devices, refer to the NI-USRP Help or to the appropriate specifications for your device:

  • USRP-2945 Specifications
  • USRP-2955 Specifications

FPGA VIs

You can view the FPGA VI in the Project Explorer window by expanding My Computer. The following FPGA VI is included:

  • Streaming 2945 2955 (FPGA).vi—Enables sample acquisition using the Rx Streaming Engine. Specify the number of samples to acquire using the rx.samples to acquire parameter.

If you have NI USRP RIO support installed, you can open the FPGA VI by navigating to My Computer»<Device Name>»Streaming 2945 2955 (FPGA).vi. You can modify the FPGA VI or create one as needed for your application.

Running this Sample Project

  1. Ensure all hardware is set up and visible in Measurement & Automation Explorer (MAX). Record the resource names of all NI 2945/2955 devices because you will need these names later.
  2. In the Project Explorer window, open a host VI, such as Rx Streaming (Host).vi.
  3. In the RIO Device pull-down menu, specify the resource name of the USRP-2945/2955 device that you identified in step 1.
  4. Click Run. The front panel displays the acquired data.
  5. Click Stop.
  6. Experiment with different values for the front panel controls and click Run. You must restart the VI for new settings to take effect.

Using LO Sharing

This sample project can configure the LOs, as well as the LO input and LO output ports of the USRP-2945/2955. You can use the LO Configuration host VI front panel control to select a configuration for RF 0 and RF 1. The possible configurations are as follows:

  • Independent—Each channel will use its own LOs. The LO input ports will be disabled.
  • Import—Both channels will use the LOs provided at the corresponding import ports. If you are using an external LO, use the Coerced Configurations tab on any host VI front panel to see what frequency LOs are required for your configuration.
  • Shared and Export—Both channels will use the same LOs from first channel. The first channel's LOs will also be exported to the LO output ports on the device back panel. NOTE: Only RF 1 can export to the LO output ports.
  • Reimport—Both channels use the same LOs from the first channel. This mode offers maximum coherence between channels since it shares the same locally generated and reimported LO signal. The LO ouptut connectors on the device back panel receive the exported LOs from the first channel. Both channels may need external amplifiers to reimport this signal through the LO input connectors. You can share this LO signal across all the channels on the device through the LO input connectors using RF splitters and amplifiers. Note: Only the RF 1 daughterboard can export to the LO output connectors.
  • Shared—Both channels will use the same LOs from first channel.

If you are using a USRP-2945/2955 to supply LOs to other USRP-2945/2955 devices, you may require power amplification. Refer to USRP-2945 Specifications or USRP-2955 Specifications for more information.

Using an External LO

This sample project can configure the USRP-2945/2955 to use an external LO in applications that require phase synchronization.

  1. Run either Rx Streaming (Host).vi or Multi-Device Rx Streaming Time (Four Channels) (Host).vi.
  2. Use the Coerced Configurations tab located on the front panel to read the values for IF1 LO and IF2 LO.
  3. Provide an LO with the frequency specified by the IF1 LO front panel indicator to the input port labeled IF1 LO IN.
  4. Provide an LO with the frequency specified by the IF2 LO front panel indicator to the input port labeled IF2 LO IN.
  5. Re-run the host VI with the LO Configuration front panel controls set to Import.

Using LO Reimport

Connections for Single Device Reimport LO Configuration

  1. Connect the USRP-2945/2955 LO OUT 1 IF2 back panel connector to an RF power amplifier IN connector using an SMA(m)-to-SMA(m) cable.
  2. Connect the RF power amplifier OUT connector to a two-way RF power splitter.
  3. Connect the two RF power splitter RF output connectors to the USRP-2945/2955 LO IN 0 IF2 and LO IN 1 IF2 back panel connectors.
  4. Repeat steps 1 through 3 using a separate RF amplifier and RF power splitter to reimport and share the IF1 signal.

Single-Device LO Reimport Connection Diagram

Example configuration may use Mini Circuits ZFRSC-123+ RF Power Splitter and PXI-5691 RF Amplifier.

Connections for Multi-Device Reimport LO Configuration

  1. Connect the USRP-2945/2955 (master) LO OUT 1 IF2 back panel connector to an RF power amplifier IN connector using an SMA(m)-to-SMA(m) cable.
  2. Connect the RF power amplifier OUT connector to a four-way RF power splitter.
  3. Connect the four RF power splitter RF output connectors to the LO IN 0 IF2 and LO IN 1 IF2 back panel connectors of all USRP-2945/2955 devices.
  4. Repeat steps 1 through 3 using a separate RF amplifier and RF power splitter to reimport and share the IF1 signal.
  5. Share a common reference and PPS signal between the USRP devices using a clock distribution device.

Multi-Device LO Reimport Connection Diagram

Example configuration may use Mini Circuits ZFRSC-4-842+ RF Power Splitter, PXI-5691 RF Amplifier and CDA-2990 Clock Distribution Device.
